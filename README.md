# Parking Occupancy Predictor

Predicts how full a parking lot will be at a given day/time, so users can check before heading out instead of guessing.

Full-stack project: Python backend (FastAPI + SQLite) with a baseline statistical prediction model, plain HTML/CSS/JS frontend.

## How it works

The model groups historical occupancy data by **day of week + hour** and predicts occupancy as the average for that bucket. No ML yet — that's a planned v2 (see Roadmap).

Given a predicted occupancy %, the API also estimates:
- **Wait time** (via occupancy breakpoints, e.g. <50% → 0 min, 90%+ → 25 min)
- **Best time to go** (the least-busy hour for that day)

## Tech stack

| Layer      | Tech |

## API

### `GET /predict?day=Monday&hour=14`
```json
{
  "occupancy_percent": 63.1,
  "wait_minutes": 3,
  "best_time": "8:00 PM"
}
```

### `GET /best-time?day=Saturday`
```json
{
  "best_hour": 9,
  "predicted_occupancy_percent": 43.1,
  "hourly_breakdown": { "0": 12.4, "1": 9.8, "...": "..." }
}
```

### `GET /days`
Returns the valid day-of-week strings.

Full interactive docs at `/docs` (Swagger UI, auto-generated by FastAPI).

## Running it locally

```bash
pip install -r requirements.txt

# Load data (pick one)
python3 parking_predictor.py setup-dev      # synthetic test data, no internet needed
python3 parking_predictor.py fetch-real     # pulls real Seattle data
python3 parking_predictor.py load-real      # cleans + loads it into SQLite

# Start the API
uvicorn main:app --reload
# → http://127.0.0.1:8000/docs
```
